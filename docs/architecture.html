How This App Works

A plain-English guide to the Factory Inventory Management System

The Big Picture

Think of this app like a restaurant. There's a front-of-house (what customers see) and a back-of-house (the kitchen).

🖥️

Frontend (The Dining Room)

Built with Vue 3 — this is everything you see and click on in the browser. It runs on port 3000.

Like the dining room of a restaurant: it takes your order (clicks & filters), sends it to the kitchen, and presents the food (data) on your plate (screen).
⚙️

Backend (The Kitchen)

Built with Python FastAPI — this processes requests and serves up data. It runs on port 8001.

Like the kitchen: it receives orders from the dining room, prepares the data (filters, calculates), and sends back exactly what was asked for.
📁

Data (The Pantry)

JSON files stored in server/data/. No database — everything loads into memory when the server starts.

Like a pantry full of pre-made ingredients: inventory items, orders, spending records, forecasts. When the server restarts, it reloads fresh from these files (no changes are saved).

How Data Flows Through the App

When you open a page or change a filter, here's what happens step by step — like placing an order at a restaurant.

1

You interact with the page

You open the app, click a page, or change a filter (like picking "Tokyo" warehouse). The filter bar at the top saves your choices.

FilterBar.vue → useFilters.js
2

The frontend asks the backend for data

The page notices the filter changed and sends a request to the backend — like handing a ticket to the kitchen. The request includes your filter choices as parameters.

api.js sends HTTP request to localhost:8001
3

The backend filters and prepares the data

FastAPI receives the request, looks through the in-memory data, removes anything that doesn't match your filters, and packages up the results.

server/main.py → apply_filters() → filter_by_month()
4

The data gets validated

Before sending the data back, FastAPI checks that everything looks right (correct fields, correct types) — like a quality check before the dish leaves the kitchen.

Pydantic models in server/main.py
5

The page updates with fresh data

The frontend receives the response and automatically re-renders the charts, tables, and numbers you see on screen. No page reload needed — it updates in place.

Vue computed properties → template re-render

Features That Work Across All Pages

These are like building-wide systems — they work the same way no matter which page you're on.

🔍 Global Filters

The filter bar at the top lets you narrow down by time period, warehouse (San Francisco, London, Tokyo), product category, and order status. When you change a filter, every page updates to match.

🌐 Language Switching

Toggle between English and Japanese. When you switch to Japanese, everything translates — labels, product names, even the currency changes from USD to JPY.

👤 User Profile

The avatar in the top-right opens a profile menu with user details and a task list. This is mock data — there's no real login system, it's just for demo purposes.

🔗 Click-to-Detail

Many items in tables are clickable. Click a product, inventory item, or backlog entry to open a popup with more details. Like clicking a folder to see what's inside.

How to Run It

Two servers need to be running at the same time — one for the frontend, one for the backend.

🚀 Quick Start (one command)

./scripts/start.sh
Starts both servers at once. Use ./scripts/stop.sh to shut them down.

🔧 Manual Start

cd server && uv run python main.py
cd client && npm run dev
Run each in a separate terminal window.

🧪 Run Tests

cd tests && uv run pytest -v
51 tests that verify all the backend endpoints work correctly. No servers need to be running.

Key Things to Know

If you remember nothing else, remember these.

💾

Nothing Saves Permanently

This is a demo app. All data lives in JSON files and loads fresh every time the backend restarts. Any changes you make (tasks, purchase orders) disappear on restart.

🔌

Two Servers, One App

The frontend (port 3000) and backend (port 8001) are separate programs that talk to each other. Both need to be running for the app to work.

🎛️

Filters Are Global

When you set a filter on one page, it stays active when you switch pages. Think of it like adjusting the thermostat — it affects the whole building, not just one room.
